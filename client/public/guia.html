1. Visão Geral do Sistema

Central dos Desmanches é um marketplace SaaS B2C2B brasileiro que conecta desmanches credenciados (ferros-velhos de peças auto) com clientes e oficinas que buscam peças originais. O modelo de negócio cobra assinatura mensal dos desmanches para acesso às solicitações dos clientes, além de uma taxa por negociação concluída.

3. Fluxo de Negócio

Fluxo Principal — Pedido de Cliente

1. Cliente cria pedido → 2. Desmanches visualizam no mural → 3. Desmanche envia proposta → 4. Cliente aceita proposta → 5. Negociação criada (chat habilitado) → 6. Desmanche marca como Enviado → 7. Cliente confirma recebimento → 8. Gate de Avaliação (prazo 7 dias) → 9. Avaliação → Negociação Concluída

Fluxo Anúncio por Desmanche (Novo)

1. Desmanche publica anúncio → 2. Clientes visualizam no marketplace → 3. Cliente clica "Comprar" → /buy → 4. Negociação criada diretamente → 5. Mesmo fluxo de envio/recebimento

Anúncios de desmanche expiram automaticamente em 72 horas se não houver compra. O desmanche pode reativar o anúncio a qualquer momento.

Fluxo de Aprovação de Desmanche

1. Cadastro com dados da empresa → 2. Upload de documentos (CNPJ, Detran, Alvará) → 3. Status: pendente → 4. Admin verifica e aprova/rejeita → 5. Desmanche ativo — acessa o painel

Gate de Avaliação

Após o cliente confirmar recebimento, um prazo de 7 dias é iniciado para avaliação. Se o prazo expirar sem avaliação, conta como atraso. Com N avaliações atrasadas (configurável pelo admin), o desmanche é bloqueado de enviar novas propostas.

4. Autenticação & Sessões

🔑 JWT Token

Token JWT armazenado em localStorage com a chave peca_rapida_token. Todas as requisições protegidas enviam o token no header Authorization: Bearer TOKEN.

👤 Tipos de Usuário

client — Clientes que buscam peças
desmanche — Empresas de desmanche credenciadas
admin — Administradores da plataforma

🔒 Middleware de Permissão

authMiddleware valida JWT.
requireType(["client"]) restringe por tipo.
Desmanches precisam de status active para operar.

🔗 Endpoints de Auth

POST /api/auth/login — clientes/admin
POST /api/auth/login-desmanche — desmanches
POST /api/auth/register — novo cliente
POST /api/auth/register-desmanche — novo desmanche

10. Integração Asaas (Pagamentos)

O Asaas é o gateway de pagamentos integrado para cobranças de assinatura dos desmanches. Suporte a Pix e Boleto Bancário.

🔧 Configuração

Arquivo: server/asaas.ts
Ambiente: ASAAS_ENVIRONMENT (sandbox / production)
API Key: ASAAS_API_KEY
URL Sandbox: https://sandbox.asaas.com/api/v3
URL Produção: https://api.asaas.com/v3

📞 Funções Implementadas

isAsaasConfigured() — verifica se API key está configurada
createAsaasCustomer() — cria cliente no Asaas
createAsaasCharge() — gera cobrança (Pix/Boleto)
getAsaasChargeStatus() — consulta status de cobrança
getDueDateString() — gera data de vencimento

🔄 Fluxo de Cobrança

1. Desmanche configura assinatura → POST /api/billing/setup
2. Sistema cria cliente no Asaas (se não existir)
3. Gera cobrança com vencimento em 3 dias
4. Retorna link de pagamento (invoice URL)
5. Webhook confirma pagamento → status active

🪝 Webhook

Endpoint: POST /api/billing/webhook
Evento: PAYMENT_CONFIRMED ou PAYMENT_RECEIVED
Ação: marca a transação como paid pelo asaasChargeId
Modo offline: funciona sem Asaas (status manual via admin)

⚠️ Modo Offline

Quando ASAAS_API_KEY não está configurada, o sistema funciona normalmente sem gerar cobranças reais. O admin pode confirmar pagamentos manualmente via painel de Assinaturas & Receitas.

11. APIs Externas Gratuitas

O sistema utiliza 4 APIs externas gratuitas, sem necessidade de chaves de API ou autenticação. Todas são chamadas diretamente do frontend (browser).

🏛️ BrasilAPI — Consulta CNPJ (Receita Federal)

📌 URL Base

https://brasilapi.com.br/api/cnpj/v1/{cnpj}

📍 Onde é usado

client/src/pages/CadastroDesmanche.tsx
Função: fetchCnpj()
Trigger: ao digitar o CNPJ no wizard de cadastro do desmanche

⚙️ Comportamento

Ao digitar um CNPJ de 14 dígitos, o sistema consulta a Receita Federal via BrasilAPI e auto-preenche todos os campos do formulário automaticamente.

📦 Campos Auto-preenchidos

razao_social → Razão Social
nome_fantasia → Nome Fantasia
ddd_telefone_1 → Telefone
logradouro, numero, complemento → Endereço
bairro, municipio, uf → Cidade/Estado
cep → CEP
qsa[0].nome_socio → Nome do Responsável
natureza_juridica, situacao_cadastral

# Exemplo de requisição (sem autenticação)
GET https://brasilapi.com.br/api/cnpj/v1/98765432000110
# Retorna: razão social, endereço, sócios, situação cadastral, atividades, etc.
📬 ViaCEP — Busca de Endereço por CEP (Correios)

📌 URL Base

https://viacep.com.br/ws/{cep}/json/

📍 Onde é usado

1. CadastroDesmanche.tsx — step de endereço no wizard de cadastro
2. ProfileTab.tsx — aba de perfil do cliente
Função: fetchCep()

⚙️ Comportamento

Ao digitar um CEP de 8 dígitos, o sistema consulta o ViaCEP e auto-preenche rua, bairro, cidade e estado. O usuário só precisa informar o número e complemento.

📦 Campos Auto-preenchidos

logradouro → Rua/Avenida
bairro → Bairro
localidade → Cidade
uf → Estado (sigla)
cep → CEP formatado

# Exemplo de requisição (sem autenticação)
GET https://viacep.com.br/ws/01310100/json/
# Retorna: {"cep":"01310-100","logradouro":"Avenida Paulista","bairro":"Bela Vista","localidade":"São Paulo","uf":"SP"}
🚗 API FIPE (Parallelum) — Marcas e Modelos de Veículos

📌 URL Base

https://parallelum.com.br/fipe/api/v1/

📍 Onde é usado

client/src/components/client/CreateOrderWizard.tsx
Funções: fetchFipeBrands() e fetchFipeModels()
Trigger: Passo 2 do wizard de criação de pedido

🔗 Endpoints Consumidos

GET /carros/marcas → marcas de carros
GET /motos/marcas → marcas de motos
GET /caminhoes/marcas → marcas de caminhões
GET /{tipo}/marcas/{codigo}/modelos → modelos de uma marca

⚙️ Comportamento

Quando o usuário seleciona carro, moto ou caminhão, o wizard busca automaticamente as marcas disponíveis na tabela FIPE. Ao escolher a marca, busca os modelos. Outros tipos (van, barco, avião etc.) usam lista estática.

# Buscar marcas de carros
GET https://parallelum.com.br/fipe/api/v1/carros/marcas
# Buscar modelos da Volkswagen (código 59)
GET https://parallelum.com.br/fipe/api/v1/carros/marcas/59/modelos
# Retorna: [{"codigo":"59","nome":"Volkswagen"}, ...] / {"modelos":[{"codigo":829,"nome":"Gol"}...]}
Tipos com FIPE integrada:
🚗 carros 🏍️ motos 🚛 caminhoes
Outros tipos usam lista estática: ônibus, van, barco, avião, helicóptero, bicicleta, trator, outro
📦 Correios — Rastreamento de Encomendas

📌 Status Atual

PARCIALMENTE INTEGRADO
O código de rastreio é armazenado no banco e exibido para o cliente, mas a consulta de status em tempo real nos Correios ainda não está implementada como chamada de API automática.

📍 Como funciona hoje

1. Desmanche digita o código de rastreio ao marcar envio (PATCH /api/negotiations/:id/ship)
2. Código é salvo em negotiations.tracking_code
3. Exibido ao cliente na aba Negociações
4. Cliente acessa os Correios manualmente para rastrear
